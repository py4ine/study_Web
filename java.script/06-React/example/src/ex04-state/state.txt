state
컴포넌트 내부에서 관리되는 데이터
컴포넌트의 동적인 데이터를 저장하고 관리하는데 사용

중요!
컴포넌트는 state가 변경되면 컴포넌트를 다시 렌더링 함
불필요한 상태값이 포함되어 있을 경우 불필요한 렌더링 발생으로 성능 저하
부모에서 자식으로 단방향 데이터 흐름을 가짐
이 흐름을 명확하게 하기 위해 state는 UI와 관련된 값만 포함하는 것을 권장

따라서 렌더링 및 데이터의 흐름에 관련 있는 값만 포함하도록 사용

특징
가변적으로 컴포넌트 자신의 state 변경
state는 다른 컴포넌트에서 접근 할 수 없고 내부에서만 관리됨
constructor()나 useState() 훅을 사용하여 state초기화 가능
클래스형 컴포넌트의 경우 setState()메서드로 state를 업데이트
객체지향에서 private멤버 변수와 비슷한 개념
